Hexo+Github搭建个人博客（Windows）

  • 个人博客搭建
技术
  view

前言

最近闲来无事，用Hexo+Github搭建了个人专属博客用来记录一些文献阅读笔记。完工之后先练练手，浅浅写个搭建记录。从零开始搭建属于自己的静态博客网站，使用Hexo博客框架部署到Github，可以在不用购买服务器的情况下拥有属于自己的博客网站。简直美滋滋！

一 准备工具

1.1 Node.js

  • 由于Hexo需要Node.js支持，因此需要先下载安装Node.js，这里直接到官网选择最新的LTS（长期支持版）即可。（安装过程就不再详细介绍了，参考相关博客即可，当然，如果有时间后续会专门写一个Node.js安装博客）。安装完成后会附带有包管理工具npm。

  • 验证是否安装成功。win+R，输入cmd打开控制台，输入以下命令:

    • node -v：显示安装的Node.js版本

    • npm -v：显示安装的npm版本

1.2 Git

  • Git简介：Git是一个开源的分布式版本控制软件,用以有效、高速的处理从很小到非常大的项目版本管理。GitHub是一个基于Git的远程文件托管平台。

  • 安装过程不再详细介绍，参考相关博客，直接到官网下载并进行安装配置。

  • 验证是否安装成功。在cmd控制台输入以下命令：

    • git --version：显示安装的Git版本

1.3 Hexo

  • 准备Hexo之前，这里先简单介绍一下。Hexo是一款快速、简洁、高效的博客框架。主要有以下几个特点：

    • 速度快：使用Node.js生成页面，每秒可以构造数百个文件。
    • 支持Markdown语法：可以使用Markdown解析渲染文章，非常方便。
    • 部署简单：只需要一个指令便可将博客提交至Github或是其他个人网页。

  • 安装Hexo

    • npm安装命令：nmp install hexo-cli -g

  • 验证是否安装成功。在cmd控制台输入以下命令：

    • hexo -v ：显示安装的Hexo版本

二 搭建仓库

  • 新建一个名为username.github.io的仓库（注意：username为Github用户名，否则无效）

三 生成SSH Keys

提交代码需要Github 权限才可以，但是直接使用用户名和密码太不安全了，所以使用ssh key 来解决本地和服务器的连接问题。

  • 生成ssh：

    • 打开命令行工具（Git Bash Here），输入命令：ssh-keygen -t rsa -C "email address"生成.ssh密匙。

    • 执行命令成功之后会在 C:\Users\用户名\目录下生成一个.ssh文件夹。打开该文件夹找到id_rsa.pub文件，用记事本打开复制里面的内容。

    • 打开github主页，进入Setting -> SSH and GPG keys -> New SSH key，把复制的内容粘贴进去，title 随便填，保存以后这样公钥就添加成功了，设置好如下图:

  • 测试ssh是否绑定成功：

    • ssh -T git@github.com

  • 配置github：

    • git config --global user.name "username"，username是用户名
    • git config --global user.email "register email"，register email是Github的注册邮箱

四 本地生成博客

  • 初始化hexo

    • 在一个路径下新建一个文件夹（名字随意）作为存放代码的地方，例如我这里是D:\Blog。

    • 在D:\Blog文件夹下右键打开命令行工具（Git Bash Here），输入命令hexo init进行初始化。

    • Hexo会自动下载一些文件到该目录下，其中_config.yml是配置文件，source文件夹存放的是博客内容的源文件（.md文件），themes文件夹下存放的是hexo主题文件。

  • 生成html文件

    • hexo g：执行该命令，会在public文件夹下生成相关的html文件（将来需要提交至Github）。

  • 本地预览

    • hexo s： 执行该命令，可以开启本地预览服务，打开浏览器访问http://localhost:4000/就可以看到博客内容了。
